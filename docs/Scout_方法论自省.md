## 一、3 个根本性方法论缺陷 (2026-04-25 一次评估)

### 缺陷 1: 从"识别方向"到"选出标的"之间的真空地带

**现状**: Scout 链条 政策动机 → 产业方向 → 行业候选 → ??? → 具体股票
6 维度评分作为过滤器,但"早期赢家"经常财务正在改善但还没到 Scout 标准。

**盲点指标 (规则层未覆盖)**:
- 毛利率拐点 (领先净利润 2-3 季度)
- 研发支出占比上升 (科技股早期信号)
- 经营现金流 / 净利润 > 1 (业绩真实性)

**修复时机**: Phase 4 (2026-06)
**修复方式**: 加进维度 4 辅助指标,与 Sonnet stage 2 配合判断


### 缺陷 2: 政策→产业链推导的链路完整性未验证

**现状**: 依赖手工录入 related_stocks + LLM 从政策文件推导
**盲点**: 技术细节级的产业链关系 (如 DUV vs EUV 节点) 无法捕获,
导致可能在错的细分赛道上选股

**修复时机**: Phase 5 (远期)
**修复方式**: 接入产业链知识图谱 (同花顺产业链/Choice 等),
加 Quality Gate 做 LLM 输出独立验证


### 缺陷 3: 动机判断的"根本主观性"未充分披露

**现状**: v1.54 做了 motivation_uncertainty 字段 (low/medium/high) +
漂移监控,但推送中没有"动机判断本身可能是个错误叙事"的提示

**风险**: 用户可能在"自己的故事"里走太深,即使动机错了也不容易察觉

**修复时机**: Phase 2C (5 月底-6 月)
**修复方式**: A 级推荐模板加"可证伪条件"——
如果 X 发生说明动机错了,这是动机判断的反证条件

## 二、5 条具体方法论改进 (2026-04-25 二次更新)

来源: 二次评估发现 Scout 维度 4 (数据验证) 和维度 5 (标的财务) 过粗。
吸收基本面价值投资派的"高质量增长"框架。

### 改进 1: 反指标硬过滤层 (优先级 🔴 高)

**当前缺陷**: Stage 1 硬底线只有 4 条 (policy_fatal/Z<1.81/gap_closed/risk_flag),
没有公司级负面过滤。一只股票如果 6 维度 A 级,即使大股东刚减持 10% 仍然推荐。

**吸收内容**: Stage 1 后 Stage 2 评分前,加一个反指标硬过滤层:
- 过去 90 天大股东减持 > 5% → 降级到 candidate
- 商誉 / 净资产 > 30% → 降级到 candidate
- 应收账款增速 - 营收增速 > 20pp 持续 2 季度 → 降级
- 解禁未来 180 天 > 15% 流通股 → 降级

**修复时机**: Phase 2B (5 月)
**优先级**: 🔴 高 (登记为 TD-012)
**预期工作量**: 1-2 天 (纯规则,不需要 LLM)


### 改进 2: 维度 4 数据验证升级 (优先级 🟡 中)

**当前缺陷**: 维度 4 几乎不看毛利率结构变化和现金流质量。

**吸收内容**:
- 毛利率变化方向 (上升/稳定/下降)
- 单位售价 vs 单位成本变化
- 经营现金流 / 净利润比率 (长期 < 0.5 警惕)
- 应收账款增速 vs 营收增速 (前者持续高于后者警惕)
- 存货增速 vs 营收增速 (同上)
- 合同负债/预收款变化 (上升通常代表订单需求强)

**修复时机**: Phase 2C (5 月底-6 月)
**预期工作量**: 3-5 天 (数据全部从 akshare 来,不需要新数据源)


### 改进 3: 产品阶段字段 (优先级 🟡 中)

**当前缺陷**: scout_range=early/active/mature 是行业级的,
没有公司级的产品阶段。同一行业里两家公司,
一家产品在验证期一家在标准化期,Scout 当前给同样分数。

**吸收内容**: related_stocks 表加 product_stage 字段,5 个枚举值:
- trial (试验期: 客户试用,收入小)
- validation (验证期: 进入头部客户,小批量)
- ramp (放量期: 大客户复购,收入加速) ← 最该投的阶段
- standard (标准化期: 产品成为行业默认选择)
- dominant (垄断/寡头期: 定价权增强)

**修复时机**: Phase 2C (5 月底-6 月)
**预期工作量**: 1-2 天 (字段定义 + 冷启动期手工录入,后期 LLM 辅助维护)


### 改进 4: 利润质量分级 (优先级 🟢 低)

**当前缺陷**: recommendations 表只看 total_score,完全没有"利润来源拆解"。
意味着 Scout 把"靠政府补贴的伪增长"和"靠定价权的真增长"按同分推荐。

**吸收内容**: 维度 4 加一条"利润质量分级":
- high: 主营毛利改善 / 费用率下降但研发增长
- medium: 资产减值减少
- low: 投资收益 / 政府补贴 / 会计变更

**修复时机**: Phase 4 (6 月)
**约束**: 必须等 TD-009 Stage 2 LLM 激活后才能做,
规则层抓不住"利润来源是什么性质",需要 LLM 读财报附注判断。


### 改进 5: capex / 订单匹配度判断 (优先级 🟢 低)

**当前缺陷**: 看 v3_capex 数据,但没有"capex 是否匹配订单"的判断。
导致可能正面评价"产能过剩前夜"的扩产潮 (锂电 2021 类似情景)。

**吸收内容**: 维度 3 加一条"capex 与订单匹配度":
- 合同负债增长率 vs 在建工程增长率
- 前者快于后者: 好信号 (有订单才扩产)
- 后者快于前者: 警惕信号 (盲目扩产前兆)

**修复时机**: Phase 4 (6 月)

## 三、明确不引入的方向

记录这些是为了未来不再被同类资料诱惑去补:

| 不引入的方向 | 原因 |
|---|---|
| CANSLIM 技术面共振 (Pocket Pivot / 200日均线 / 杯柄形态) | 与 Scout 早期识别定位冲突,这些信号出现时已经晚 |
| 欧奈尔"业绩拐点+技术突破"入场时机 | Scout 在欧奈尔说"还不能买"时就该建仓 |
| "30-40% 失败率 + 止损纪律"框架 | 是短中期交易者语言,与 3-5 年持有冲突 |
| 动量信号 (相对强势/突破时跟随) | v1.55 论证过会让 Scout 漏掉早期机会 |
| 短期股价/成交量行为 | Scout 看 3-5 年,不应把短期行为当输入 |
| Demo 模式 (v1.64) | Scout 私用非分享,永远不做 |
| LLM 预填模式 (v1.66) | 同上 |

## 五、归档惯例 (Scout 三角色模型缺失环节的补救)

**问题**: Scout 三角色模型 (Claude Code 执行 + 外部 LLM 辅助大脑 + 用户决策)
产生的判断性结论没有自动归档机制。代码改动有 git history,
但"X 是 Scout 盲点"这种判断性结论没有载体,半年后会丢失。

**惯例 (从此开始执行)**:
当与外部 LLM (我或其他模型) 讨论中产生重要判断性结论时,
追加到本文档相应章节,或新增章节。每条结论必须包含:
- 日期
- 结论
- 理由
- 适用范围/时机
- 反决定的触发条件 (什么情况下推翻这个结论)

**已登记为 TD-011** (判断性结论归档惯例尚未自动化)
**修复时机**: 不阻塞,流程性 TD

## 二、3 层根本认知突破

### 第一层 (上午发现): 评分体系结构性偏差

Scout 6 维度里 4 个维度 (d1+d2+d3+d4 偏向) 结构性偏向政策驱动, 科研/宏观维度权重 ~0%。

**实证**: 5 天产出 95 条独立判断, A 级 3 只全部半导体设备, 完全错过 2024-2025 AI 链 (中际旭创+1400% / 胜宏+900% / 工业富联+300%)。

详见: [docs/Scout_设计意图与实现偏离.md](Scout_设计意图与实现偏离.md) 第一-七节

### 第二层 (下午发现): 缺乏自省能力

Scout 错过 AI 链 2 年, 但 Scout 不知道自己错过了什么。没有 "unknowns 表" / "覆盖范围 vs 市场" / "错过机会复盘" 机制。

详见: [docs/Scout_设计意图与实现偏离.md](Scout_设计意图与实现偏离.md) 第八节

### 第三层 (晚上发现): 部分偏离不是设计错, 是代码未实装

完整蓝图盘点 47 项发现:

- 三市场差异权重 (TD-020): 蓝图明示 US 财务权重 ×2, 代码同一套权重
- mixed_subtype 下游分级 (TD-021): v1.60 设计未实装
- mode 切换 + 前视偏差 (TD-022): v1.56 设计未实装

**关键转折**: 评分体系修复方向从 "Layer 2 全面重写" 修订为 "Layer 1 实装已有设计可能阶跃式提升, Layer 2 后续再评估"。

详见: [docs/Scout_设计意图与实现偏离.md](Scout_设计意图与实现偏离.md) 第五A 节

## 五、关键事实存档 (避免散落)

### 5.1 RecommendationAgent 实时读 watchlist (不需重启)

证据: `agents/recommendation_agent.py` 9 处 SELECT, 0 处 cache.
意义: 补字段 / 改 watchlist 数据立即生效, 不需 restart scout serve PID.

### 5.2 d4 weighted 计算公式

V1/V3/D4 90 天信号数 → 100 满分, weight 0.10.
实测: 002371/688012 都是 16 条信号 → score=100, weighted=10.0.

### 5.3 GateAgent verify 加分阶梯

同行业信号 11 条 → +5 / 17 条 → +5 (≥10 条阶梯式加 5).

### 5.4 gap_fillability 0-5 → d3 score mapping

`{0:0, 1:25, 2:50, 3:75, 4:100, 5:100, NULL:50}`

注意: 4 和 5 在 scoring 等价 (TD-015), 0 和 1 触发 Stage 1 fatal.

### 5.5 cmd_recommend CLI 单股模式 industry resolution bug

direct call `analyze(symbol)` 时 industry=null, 走默认值, 分数无意义 (例: 002371 单股调用 46.25, 但全量评分 90.94).

绕过: 设置 `SCOUT_DB_PATH` env 强制指向 production DB.
登记为 [TD-016](Scout_技术债务清单.md#td-016--cli-单股模式-industry-resolution-失效).

## 八、新会话入场机制 (本日最后产出)

为解决 "换聊天信息丢失" 问题, 建立入场机制。

### 机制设计原则

- **不依赖 claude.ai (我) 的记忆** (它每次会话重置)
- **不过度依赖 userMemories** (压缩损失大)
- **依赖 Claude Code + git docs/** (持久化稳定)

### 入场流程

1. 用户在新会话提 Scout → Claude Code 触发入场仪式
2. 读 5 份 docs + 4 份 memory + git log
3. 检查 Scout serve 状态
4. 用 SNAPSHOT 模板报告给用户
5. 等用户指令, **不主动执行**

### 实施位置

- [docs/Scout_新会话入场.md](Scout_新会话入场.md) — 5 步流程详细文档
- [CLAUDE.md](../CLAUDE.md) 顶部 banner — 触发引用, 任何 Claude Code 会话第一眼看到
- 本节作为今日认知日的 closing — 标记此机制建于 2026-04-25 末尾

### 关键认知

未来 Claude.ai 端不需要 "记得" 今天发生了什么。Claude Code 通过 SNAPSHOT 喂上下文, claude.ai 端读 SNAPSHOT 即可获得和今天同等质量的判断能力。**Scout 的"长期记忆"在 git docs/, 不在任何 LLM 的会话状态里。**
